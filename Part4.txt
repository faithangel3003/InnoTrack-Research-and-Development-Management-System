8. Notifications

7. BUSINESS RULES TO ENFORCE
   - Users can only read messages in channels they are members of
   - Soft-delete messages: replace content with
     "This message was deleted" — keep record in DB
   - Edited messages must show IsEdited = true with timestamp
   - Direct message channels are auto-created between 2 users
     (check for existing before creating)
   - Announcements with ExpiresAt in the past are
     auto-hidden from listings
   - Notifications auto-generated for:
     → Task assignment
     → @mention in a message (parse @username in content)
     → New announcement posted
     → Project status change
     → Document uploaded to a project
     → Milestone completed
   - Typing indicators expire after 3 seconds server-side
   - Unread message count per channel tracked via
     ChannelMembers.LastReadAt
   - Message attachments use same
     IFileStorageService from Documents module

⚙️ SIGNALR CONFIGURATION (Program.cs additions)
═══════════════════════════════════════════════
- Register SignalR: builder.Services.AddSignalR()
- Map hub: app.MapHub<CollaborationHub>("/hubs/collaboration")
- Configure JWT auth for SignalR:
  (token passed via query string for WS connections)
  options.Events = new JwtBearerEvents {
    OnMessageReceived = context => {
      var accessToken = context.Request.Query["access_token"];
      var path = context.HttpContext.Request.Path;
      if (!string.IsNullOrEmpty(accessToken) &&
          path.StartsWithSegments("/hubs/collaboration"))
        context.Token = accessToken;
      return Task.CompletedTask;
    }
  };
- Blazor client: install
  Microsoft.AspNetCore.SignalR.Client NuGet package
