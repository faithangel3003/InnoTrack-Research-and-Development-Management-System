Build the complete "Research Documentation & File Management" module for:
"InnoTrack: Research and Development Management System for Innovation and Collaboration"

This module builds on top of the existing modules:
- User & Role Management (Users, Roles, Organizations, JWT Auth)
- Project Tracking (Projects, Tasks, ProjectMembers)
Assume all previous models, DbContext, repositories,
services, and auth middleware already exist.

═══════════════════════════════════════════════
🛠️ TECH STACK
═══════════════════════════════════════════════
- Backend:     ASP.NET Core Web API (C#)
- Frontend:    Blazor Web App (.NET 8)
- Database:    MySQL (via XAMPP / phpMyAdmin, local dev)
- ORM:         Entity Framework Core with Pomelo.EntityFrameworkCore.MySql
- Auth:        ASP.NET Core Identity + JWT Bearer Tokens
- Pattern:     Repository Pattern + Service Layer + DTOs
- File Storage: Local file system (wwwroot/uploads/) for dev,
                abstracted via IFileStorageService for
                future cloud migration
- API Style:   RESTful

═══════════════════════════════════════════════
👤 ROLE PERMISSIONS FOR THIS MODULE
═══════════════════════════════════════════════
- SystemAdmin    → View all documents, manage categories/tags
- ProjectManager → Upload, edit, delete documents within
                   own projects; manage document categories
- TeamMember     → Upload documents to assigned projects;
                   view/download documents in their projects;
                   cannot delete others' documents

═══════════════════════════════════════════════
🗄️ DATABASE TABLES (MySQL via XAMPP)
═══════════════════════════════════════════════
Generate EF Core migrations for the following tables:

1. Documents
   - Id (PK, GUID)
   - Title (string, required)
   - Description (text, nullable)
   - FileName (string – stored file name on disk)
   - OriginalFileName (string – original uploaded name)
   - FilePath (string – relative path in wwwroot/uploads/)
   - FileSize (long – bytes)
   - FileType (string – MIME type e.g. application/pdf)
   - FileExtension (string – .pdf, .docx, .xlsx, etc.)
   - ProjectId (FK → Projects, nullable)
   - CategoryId (FK → DocumentCategories, nullable)
   - UploadedByUserId (FK → Users)
   - OrganizationId (FK → Organizations)
   - Version (int, default 1)
   - IsArchived (bool, default false)
   - CreatedAt, UpdatedAt

2. DocumentVersions
   - Id (PK, GUID)
   - DocumentId (FK → Documents)
   - VersionNumber (int)
   - FileName (string)
   - FilePath (string)
   - FileSize (long)
   - UploadedByUserId (FK → Users)
   - ChangeNotes (text, nullable)
   - CreatedAt

3. DocumentCategories
   - Id (PK, int)
   - Name (string, required)
   - Description (string)
   - OrganizationId (FK → Organizations)
   - CreatedAt

4. DocumentTags
   - Id (PK, int)
   - Name (string, required)
   - OrganizationId (FK → Organizations)

5. DocumentTagMap (junction table)
   - DocumentId (FK → Documents)
   - TagId (FK → DocumentTags)
   - PRIMARY KEY (DocumentId, TagId)

6. DocumentAccessLogs
   - Id (PK, GUID)
   - DocumentId (FK → Documents)
   - UserId (FK → Users)
   - Action (enum: Viewed, Downloaded, Uploaded,
                   Updated, Deleted, Archived)
   - AccessedAt (DateTime)
   - IPAddress (string)

Use connection string:
"Server=localhost;Port=3306;Database=innotrack_db;User=root;Password=;"

   IFileStorageService / LocalFileStorageService
   Methods:

   → DeleteFileAsync(string filePath)

6. CONTROLLERS (REST API)
   DocumentsController.cs
   → GET    /api/documents                    [All roles]
   → GET    /api/documents/{id}               [All roles]
   → GET    /api/documents/{id}/download      [All roles]
   → GET    /api/documents/{id}/versions      [All roles]
   → GET    /api/projects/{id}/documents      [All roles]
   → POST   /api/documents/upload             [PM, TeamMember]
     (multipart/form-data)
   → PUT    /api/documents/{id}               [PM, TeamMember owner]
   → POST   /api/documents/{id}/version       [PM, TeamMember owner]
   → PATCH  /api/documents/{id}/archive       [PM, SystemAdmin]
   → DELETE /api/documents/{id}               [PM, SystemAdmin, SuperAdmin]
   → GET    /api/documents/search?q=&tags=
            &category=&projectId=             [All roles]

   DocumentCategoriesController.cs
   → GET    /api/document-categories          [All roles]
   → POST   /api/document-categories          [SystemAdmin, PM]
   → PUT    /api/document-categories/{id}     [SystemAdmin, PM]
   → DELETE /api/document-categories/{id}     [SystemAdmin]

   DocumentTagsController.cs
   → GET    /api/document-tags                [All roles]
   → POST   /api/document-tags                [SystemAdmin, PM]
   → DELETE /api/document-tags/{id}           [SystemAdmin]

   DocumentAccessLogsController.cs
   → GET    /api/documents/{id}/access-logs   [SystemAdmin, SuperAdmin, PM]

7. BUSINESS RULES TO ENFORCE
   - File size must not exceed 50MB
   - Only allowed file types accepted (validate MIME + extension)
   - TeamMember can only delete their own uploaded documents
   - ProjectManager can delete any document within their project
   - Uploading a new version keeps the old file in DocumentVersions
   - Archiving a document hides it from listings but keeps the file
   - Every view/download/upload/delete must be logged to
     DocumentAccessLogs
   - File paths must NEVER be exposed directly in API responses
     (use /api/documents/{id}/download endpoint instead)
   - Documents without a ProjectId belong to the organization globally

✅ REQUIREMENTS & NOTES
═══════════════════════════════════════════════
- Never expose raw file paths in API responses
- Always serve files through the download endpoint
- Validate file type by MIME type AND extension server-side
- Max file size configurable via appsettings.json:
  "FileStorage": { "MaxFileSizeMB": 50,
  "UploadPath": "wwwroot/uploads" }
- Blazor upload must use multipart/form-data with
  HttpClient (not standard EditForm)
- Show upload progress using
  IProgress<long> or StreamContent
- All file access events logged to DocumentAccessLogs
- Soft-delete only: archived docs kept on disk
- Hard-delete removes DB record AND file from disk
- Return proper HTTP status codes:
  200, 201, 400, 401, 403, 404, 413 (file too large)
- MySQL XAMPP connection:
  Server=localhost;Port=3306;
  Database=innotrack_db;User=root;Password=;
