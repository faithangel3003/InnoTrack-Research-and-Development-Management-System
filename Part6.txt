System-Wide Security Module Prompt

You are an expert ASP.NET Core and Blazor developer
specializing in enterprise application security.

Implement the complete security layer for:
"InnoTrack: Research and Development Management System
for Innovation and Collaboration"

This security module wraps ALL existing modules:
- User & Role Management
- Project Tracking & Monitoring
- Research Documentation & File Management
- Collaboration & Communication
- Innovation Analytics & Reports

Assume all previous models, DbContext, repositories,
services, JWT auth, and RBAC middleware already exist.

entire system without replacing existing logic.

🔑 1. PASSWORD POLICY
═══════════════════════════════════════════════

Rules to enforce:
- Minimum length: 12 characters
- Must contain: uppercase (A-Z), lowercase (a-z),
                numbers (0-9)
- No whitespace allowed
- Hashing: BCrypt with 12 rounds
- Never store plain text passwords anywhere
- Passwords never returned in any API response or DTO

Generate:

A. PasswordPolicyValidator.cs
   (reusable static class)
   Methods:
   → Validate(string password)
     Returns: ValidationResult
     (IsValid: bool, Errors: List<string>)
   Rules checked:
   - Length >= 12
   - Contains uppercase letter [A-Z]
   - Contains lowercase letter [a-z]
   - Contains digit [0-9]
   - No whitespace characters
   - Not null or empty

B. PasswordHashService.cs
   Interface: IPasswordHashService
   Methods:
   → string Hash(string plainPassword)
      Uses BCrypt.Net.BCrypt.HashPassword(
        password, workFactor: 12)
   → bool Verify(string plainPassword,
                  string hashedPassword)
      Uses BCrypt.Net.BCrypt.Verify()
   → bool NeedsRehash(string hashedPassword)
      Checks if hash uses less than 12 rounds

C. Enforce in:
   - CreateUserDto.cs:
     Add [PasswordPolicy] custom validation attribute
   - AuthService.cs:
     Call PasswordPolicyValidator before hashing
   - UserService.cs (ChangePassword):
     Validate → Hash → Store
   - Never include PasswordHash field in any
     response DTO

D. PasswordPolicyAttribute.cs
   (Custom ValidationAttribute)
   - Calls PasswordPolicyValidator.Validate()
   - Returns combined error message if invalid
   - Used as: [PasswordPolicy] on DTO properties

🔒 2. DATA HANDLING & ENCRYPTION POLICY
═══════════════════════════════════════════════

A. AES-256-CBC ENCRYPTION SERVICE
   IEncryptionService / AesEncryptionService.cs

   Config in appsettings.json:
   "Encryption": {
     "Key": "<32-byte base64 key>",
     "IV":  "<16-byte base64 IV>"
   }

   Methods:
   → string Encrypt(string plainText)
      - Generate random IV per encryption
      - Use AES-256-CBC
      - Return Base64(IV + CipherText)
   → string Decrypt(string cipherText)
      - Extract IV from first 16 bytes
      - Decrypt remainder
      - Return plain text
   → string GenerateSecureKey()
      - Uses RandomNumberGenerator
      - Returns 32-byte Base64 string

   Use for:
   - Session tokens stored in DB
   - Sensitive config values in DB
   - Any token stored outside JWT

B. COOKIE SECURITY CONFIGURATION
   In Program.cs:
   builder.Services.ConfigureApplicationCookie(
   options => {
     options.Cookie.HttpOnly = true;
     options.Cookie.SecurePolicy =
       CookieSecurePolicy.Always;
     options.Cookie.SameSite = SameSiteMode.Strict;
     options.Cookie.Name = "InnoTrack.Session";
     options.ExpireTimeSpan = TimeSpan.FromHours(8);
     options.SlidingExpiration = true;
   });

   Add Anti-Forgery token:
   - builder.Services.AddAntiforgery()
   - Add [ValidateAntiForgeryToken] to all
     state-changing Blazor forms

C. DATABASE CONNECTION SECURITY
   Connection string with SSL support:
   "Server=localhost;Port=3306;
    Database=innotrack_db;User=root;Password=;
    SslMode=Required;AllowPublicKeyRetrieval=true;"

   Note: For local XAMPP dev, SslMode=None is
   acceptable. For MonsterASP production:
   SslMode=Required

D. SENSITIVE DATA MASKING SERVICE
   IDataMaskingService / DataMaskingService.cs

   Methods:
   → string MaskEmail(string email)
      Input:  "john.doe@example.com"
      Output: "j***@example.com"
      Logic:  Keep first char of local part,
              replace rest before @ with ***,
              keep domain

   → string MaskPhoneNumber(string phone)
      Input:  "09171234567"
      Output: "--1234"
      Logic:  Show only last 4 digits
              prefixed with --

   → string MaskSssNumber(string sss)
      Input:  "123-45-6789"
      Output: "***-**-6789"
      Logic:  Mask all but last 4 digits,
              keep format separators

   → string MaskAddress(string address)
      Input:  "123 Rizal Street, Davao City"
      Output: "*** *** Street, Davao City"
      Logic:  Mask house number and street name,
              keep city/province

   → string MaskGeneric(string value, int visibleChars)
      General purpose masker

   Apply masking:
   - All User response DTOs use masked values by default
   - Create separate UnmaskedUserDto for authorized access

E. DATA UNMASKING SYSTEM
   Generate full unmasking flow:

   1. UnmaskRequest Model (DB Table)
      - Id (PK, GUID)
      - RequestedByUserId (FK → Users)
      - TargetUserId (FK → Users)
      - FieldName (string – "email", "phone", etc.)
      - VerificationToken (string – encrypted)
      - TokenExpiry (DateTime – 5 minutes)
      - IsUsed (bool)
      - CreatedAt (DateTime)

   2. UnmaskLog Model (DB Table)
      - Id (PK, GUID)
      - UserId (FK → Users – who unmasked)
      - TargetUserId (FK → Users – whose data)
      - FieldName (string)
      - IPAddress (string)
      - UserAgent (string)
      - Timestamp (DateTime)

   3. IUnmaskService / UnmaskService.cs
      → RequestUnmask(userId, targetUserId, fieldName)
         - Verify requesting user's password first
         - Generate secure 6-char token
         - Encrypt token, store with 5-min expiry
         - Returns token to client
      → VerifyAndUnmask(token, fieldName, targetUserId)
         - Validate token not expired, not used
         - Mark token as used
         - Log unmask action to UnmaskLog
         - Return actual unmasked value
      → GetUnmaskLogs(filter) – Admin only

   4. UnmaskController.cs
      → POST /api/unmask/request
         Body: { targetUserId, fieldName, password }
         Returns: { token, expiresAt }
         [Authorize – any role]
      → POST /api/unmask/verify
         Body: { token, targetUserId, fieldName }
         Returns: { unmaskedValue }
         [Authorize – any role]
      → GET  /api/unmask/logs
         [SuperAdmin, SystemAdmin only]

A. MIDDLEWARE: RoleAuthorizationMiddleware.cs

   (Custom ActionFilter Attribute)
   - Applied to sensitive config endpoints:
     [AdminReAuth] on controller/action

   - On success: allows request to proceed
   - Used on:
     → DELETE /api/users/{id}
     → PATCH  /api/users/{id}/deactivate
     → All /api/system-settings endpoints
     → POST   /api/roles (create/modify roles)

🚫 4. XSS PREVENTION
═══════════════════════════════════════════════

A. INPUT SANITIZATION SERVICE
   IInputSanitizationService / InputSanitizationService.cs
   Uses: HtmlSanitizer NuGet package

   Methods:
   → string SanitizeHtml(string input)
      - Strips all HTML tags and attributes
      - Removes script, iframe, object tags
      - Encodes special chars:
        < → &lt;  > → &gt;
        & → &amp; " → &quot;
        ' → &#x27;

   → string SanitizePlainText(string input)
      - Trims whitespace
      - Removes null bytes (\0)
      - Encodes HTML entities
      - Strips control characters

   → string SanitizeFileName(string fileName)
      - Removes path traversal: ../ ..\
      - Strips special chars except . - _
      - Enforces max length: 255 chars
      - Returns safe file name

   → T SanitizeDto<T>(T dto)
      - Reflects over all string properties
      - Applies SanitizePlainText to each
      - Returns sanitized DTO

B. GLOBAL SANITIZATION FILTER
   SanitizationActionFilter.cs
   (IActionFilter – registered globally)
   - Intercepts all incoming POST/PUT/PATCH requests
   - Calls SanitizeDto<T> on request body
   - Applied automatically via:
     builder.Services.AddControllers(options => {
       options.Filters.Add<SanitizationActionFilter>();
     });

C. CONTENT SECURITY POLICY MIDDLEWARE
   ContentSecurityPolicyMiddleware.cs
   - Adds HTTP security headers to all responses:
     Content-Security-Policy:
       default-src 'self';
       script-src 'self';
       style-src 'self' 'unsafe-inline';
       img-src 'self' data:;
       font-src 'self';
       connect-src 'self';
       frame-ancestors 'none';
     X-Content-Type-Options: nosniff
     X-Frame-Options: DENY
     X-XSS-Protection: 1; mode=block
     Referrer-Policy: strict-origin-when-cross-origin
     Permissions-Policy: geolocation=(),
                          microphone=(),
                          camera=()

D. BLAZOR-SIDE XSS PREVENTION
   - Never use @((MarkupString)userContent)
     unless sanitized first
   - Create SafeMarkup.cs helper:
     → MarkupString ToSafeHtml(string input)
        Sanitizes then wraps in MarkupString
   - All user-generated content rendered through
     SafeMarkup helper in Blazor components

💉 6. SQL INJECTION PREVENTION
═══════════════════════════════════════════════

A. EF CORE PARAMETERIZATION (Default Protection)
   - ALL database queries use EF Core LINQ
     or parameterized raw SQL only
   - NEVER string-interpolate user input
     into SQL queries
   - For raw queries use ONLY:
     context.Database.ExecuteSqlRaw(
       "SELECT * FROM Users WHERE Email = {0}",
       email)   ← parameterized
   - NEVER:
     context.Database.ExecuteSqlRaw(
       $"SELECT * FROM Users WHERE Email = '{email}'")
     ← FORBIDDEN — SQL injection risk

B. SqlInjectionValidationFilter.cs
   (IActionFilter – registered globally)
   - Scans all string inputs in query params,
     route values, and request body for
     known SQL injection patterns:
     Patterns detected (regex):
     → ' OR '1'='1
     → ; DROP TABLE
     → UNION SELECT
     → -- (SQL comment)
     → xp_cmdshell
     → EXEC(
     → CAST( ... AS
     → WAITFOR DELAY
     → 0x (hex encoding)
   - On detection:
     → Returns 400 Bad Request
     → Logs SecurityEvent with
       Severity: High, Type: SuspiciousActivity
     → Records IP, UserAgent, endpoint, payload

C. SearchQuerySanitizer.cs
   - Applied to all ?q= and ?search= query parameters
   - Strips SQL metacharacters:
     ' " ; -- /* */ xp_ sp_
   - Enforces MaxLength(200) on search input
   - Encodes remaining special characters

D. DbContext Query Hardening
   In InnoTrackDbContext.cs add:
   - Disable lazy loading:
     options.UseLazyLoadingProxies(false)
   - Enable sensitive data logging ONLY in
     Development environment
   - Add query timeout:
     options.CommandTimeout(30)
   - Add interceptor for query logging:
     QueryLoggingInterceptor.cs
     → Logs all queries longer than 2 seconds
     → Logs queries in non-production environments
       (without parameter values in production)

⚡ 8. RATE LIMITING
═══════════════════════════════════════════════

Program.cs Rate Limiting Configuration:

builder.Services.AddRateLimiter(options => {

  // Login endpoint: 5 attempts per 15 minutes per IP
  options.AddFixedWindowLimiter(
    "LoginPolicy", opt => {
      opt.PermitLimit = 5;
      opt.Window = TimeSpan.FromMinutes(15);
      opt.QueueProcessingOrder =
        QueueProcessingOrder.OldestFirst;
      opt.QueueLimit = 0;
  });

  // API general: 100 requests per minute per user
  options.AddFixedWindowLimiter(
    "ApiPolicy", opt => {
      opt.PermitLimit = 100;
      opt.Window = TimeSpan.FromMinutes(1);
  });

  // File upload: 10 uploads per minute per user
  options.AddFixedWindowLimiter(
    "UploadPolicy", opt => {
      opt.PermitLimit = 10;
      opt.Window = TimeSpan.FromMinutes(1);
  });

  // Unmask requests: 3 per 5 minutes per user
  options.AddFixedWindowLimiter(
    "UnmaskPolicy", opt => {
      opt.PermitLimit = 3;
      opt.Window = TimeSpan.FromMinutes(5);
  });

  // Report generation: 5 per hour per user
  options.AddFixedWindowLimiter(
    "ReportPolicy", opt => {
      opt.PermitLimit = 5;
      opt.Window = TimeSpan.FromHours(1);
  });

  options.OnRejected = async (context, token) => {
    context.HttpContext.Response.StatusCode = 429;
    // Log rate limit hit as SecurityEvent
    await context.HttpContext.Response
      .WriteAsync("Too many requests.", token);
  };
});

Apply policies:
[EnableRateLimiting("LoginPolicy")]
→ POST /api/auth/login

[EnableRateLimiting("UploadPolicy")]
→ POST /api/documents/upload

[EnableRateLimiting("UnmaskPolicy")]
→ POST /api/unmask/request

[EnableRateLimiting("ReportPolicy")]
→ POST /api/reports/generate

[EnableRateLimiting("ApiPolicy")]
→ Applied globally as default
